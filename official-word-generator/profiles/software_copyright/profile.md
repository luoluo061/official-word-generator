# Profile: software_copyright

## 中文名称

软著说明书

## 用途

用于软件著作权说明书、系统说明书、用户手册、操作说明、技术说明等长文档。

## 默认处理逻辑

1. 通常需要封面、目录、正文、图表和页码。
2. 目录默认建议覆盖一级和二级标题；如果后续软著模板要求三级目录，再启用 `toc.level_profile` 和 `heading.level_3`。
3. 图片、图题、表格、表题是常见内容，生成后需要重点检查图片尺寸和分页效果。
4. 当前阶段没有绑定正式软著模板，先使用占位说明；不要把一般公文模板误称为软著正式模板。

## 当前实现状态

当前是设计 profile。目录、图片、表格、正文生成能力可复用现有脚本；封面、软著专属模板、三级目录、图表编号和专属校验仍需后续实现。
